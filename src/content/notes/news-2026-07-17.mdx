## AWS CloudFrontで世界規模の障害、PayPay・note・ニコニコ生放送などに影響

米国時間7月16日午後5時40分ごろ（日本時間）、AWSのCDNサービス「CloudFront」で障害が発生し、PayPay・note・ニコニコ生放送・はてなブログなど国内の複数サービスで接続しづらい状態が続いた。

クラウド・大規模障害案件。原因はCloudFrontの「VPC Origins」機能に関するもので、この機能を経由するオリジンでエラー率が上昇し504 Gateway Timeoutなどが多発した。標準的なS3オリジンなど他の構成には影響がなかったという。AWSは同日午後9時20分ごろに復旧を発表した。マイナポータルのログイン障害も同時間帯に発生しており、関連が指摘されている。

## Thinking Machines、オープンウェイト基盤モデル「Inkling」を公開

元OpenAI CTOのミラ・ムラティ氏が率いるThinking Machines Labは7月15日、テキスト・画像・音声・動画を扱うマルチモーダルなオープンウェイト基盤モデル「Inkling」を初公開した。

## 今日のTips: NestJS の Swagger CLIプラグインで、DTOのデコレータ記述を大幅に省略できる

NestJSで<Mono>@nestjs/swagger</Mono>を使ってOpenAPIドキュメントを生成する際、素朴に書くとDTOの全プロパティに<Mono>@ApiProperty()</Mono>を手作業で付け続けることになり、フィールドが増えるほど記述量がかさむ。

公式ドキュメントによると、<Mono>nest-cli.json</Mono>にSwagger CLIプラグインを設定しておくと、ビルド時にコンパイラがDTOの型情報を解析し、デコレータを自動生成してくれる。

```json
{
  "compilerOptions": {
    "plugins": [
      {
        "name": "@nestjs/swagger",
        "options": { "introspectComments": true }
      }
    ]
  }
}
```

具体的には、<Mono>@ApiHideProperty()</Mono>が付いていない限り全プロパティに<Mono>@ApiProperty</Mono>相当の情報を自動付与し、<Mono>name?: string</Mono>のような疑問符から<Mono>required: false</Mono>を、TypeScriptの型から<Mono>type</Mono>や<Mono>enum</Mono>を、代入されたデフォルト値から<Mono>default</Mono>を、それぞれ推論する。<Mono>classValidatorShim</Mono>を有効にすればclass-validatorのデコレータからバリデーションルールも読み取る。

<Tip label="▶ 補足">
<Mono>introspectComments</Mono>を有効にすると、プロパティ上のJSDocコメントを説明文やサンプル値としてスキーマに反映できる。手書きの<Mono>@ApiProperty</Mono>デコレータをさらに減らせる。
</Tip>
