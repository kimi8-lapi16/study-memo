## テクノロジ系

### 基礎理論

- バックキャスティング
- 未来の理想像から逆算して現在取るべき行動を考える思考法。フォアキャスティング（現状から未来を予測）と対比される。

- **ペトリネット** プレース（状態）、トランジション（事象）、トークン（印）の3つの要素を使用した有向グラフでシステムの動作を記述する図法、並行する処理同士の制御の流れや同期のタイミングを分析、設計するために用いられる

- **エラー検出・訂正技術**
- **奇数パリティ** データを構築するビット全体の中でビット1の数が奇数になるようにパリティビットを付加する方式、1ビットの誤りを検出できる

- **水平パリティ** データの水平方向を対象としてパリティビットを付加する方式。垂直方向と組み合わせた垂直水平パリティチェックでは1ビットの誤り位置を特定し訂正できるが、水平パリティだけでは1ビットの誤り検出しかできない

- **チェックサム** データの合計値を検査用に付加し、誤りが発生しているか検査する方式

- **ハミング符号** 情報ビットに検査ビットを付加することで2ビットまでの誤りを検出し、1ビットの誤りを自動訂正することができる方式

- **データ構造・アルゴリズム**
- **リスト構造** 隣接するデータ同士をポインタで連結して表現する。headとtailという頭、ケツがすぐに参照できるのもポイント

- **AVL木** 2分探索木に加え「左部分木と右部分木の差が1以下」となる木。木の偏りが生じにくく探索回数を安定させやすい

- ヒープソート
- 親の値 `<=` 子の値または親の値 `>=` 子の値の関係を持つ順序木として表現し、整列後の根の値を取り出すことを繰り返して整列を行う

- クイックソート
- 中間的な基準値を求めて、それよりも大きな値を集めた区分と、小さな値を集めた区分に要素を振り分ける。次にそれぞれの区分のなかで同様な処理を繰り返す

- バブルソート
- 隣り合う要素を比較して、大小の順が逆であれば入れ替える操作を繰り返す

- **符号化・変調技術**
- **PWM（Pulse Width Modulation）** パルス幅変調 信号の強度は一定のまま、パルス信号を出力する時間を長くしたり短くしたりすることで電流・電圧を制御する方式、インバータの制御方式として用いられる

- **PAM（Puls Amplitude Modulation）** パルス振幅変調 信号の強度によって制御する方式

- Xの補集合-R（つまるところXである確率かな） 7の法則
- ある値を定期的に観測している場合に、同じ結果が7回連続で記録された時には、偶然ではなく何らかの原因があると見なす品質管理(2の7乗が128なので、1/100以下の事象が見られるなら、何かしら原因がありそうという判断軸)

### コンピュータシステム

- **メモリ管理**
- **メモリインターリーブ** 物理上は一つである主記憶領域を同時アクセス可能な複数の論理的な領域に分け、それぞれのバンクに対してデータの読み書きを並列で行うことにより、メモリアクセスの高速化を図る技術。主記憶に並列にアクセスするときはメモリインターリーブ

- **書き込み方式**
- **Write through（ライトスルー方式）** 書き込み時に主記憶とキャッシュメモリ書き込む（遅いけど整合性は取れてる)

- **Write back（ライトバック方式）** 書き込み時にキャッシュメモリにのみ書き込んでおいて、キャッシュから追い出される時に主記憶に書き込まれる(高速だが整合性が取れないため余計な制御がいる)

- メモリコンパクション
- フラグメンテーションなデータを連続領域にまとめる

- とびとびで使いにくいところを整理するイメージ

- **CPU関連**
- **コンピュータ構成要素**
- プログラムカウンター
- CPUの制御装置内にあるレジスタの一つで、次に読み出して実行すべき命令が格納されている主記憶上のアドレスを保持する役割を担う

- スタックポインタ
- スタックの最上段のアドレスを保持するレジスタで、スタック内で最後に参照されたアドレスを保持している

- サブルーチン呼び出し時に戻り先アドレス、レジスタ内容などを格納するメモリのアドレス

- ステータスレジスタ
- 割込みの許可状態、および条件分岐の判断に必要な演算結果の状態

- 命令レジスタ
- メモリから読み出された機械語命令

- ウェアラベリング
- 電子記録媒体において全体的にどのブロックも均等に使うように物理的な書き込み位置を制御すること

- **CPI（Cycles Per Instruction）** CPUが1命令を実行するのに必要とするクロック数、`CPUクロック数 x CPI`で1命令を処理するのに要する時間がわかる→性能がわかる

- **DMAコントローラー（Direct Memory Access）** 入出力装置がCPUを介さずにメモリとの間で直接データの転送を行う方式

- **ウォッチドッグタイマ** タイマには適切な時間がセットされていて、システムが正常な時は一定間隔でタイマの経過時間をリセットしているが、異常が怒るとタイマがリセットされない。設定された時間が経過するとノンマスカブル割り込みを発生させ、異常を通知する（ノンマスカブル割り込みは発生を抑制できない割り込み、逆にマスカブルは抑制可）

- **リアルタイムシステム** RTOS（Real Time OS）によって管理され、設定された時間内に処理が完了することが求められる

- **システム統合・仮想化**
- **サーバコンソリデーション** 仮想化技術を用いることで、複数の専用サーバに分散・非効率化していたアプリケーションや機能を1台(または少数)の物理サーバに整理統合すること

- **性能評価**
- **コンピュータの性能評価**
- **シミュレーションを用いた方法** モンテカルロ、離散イベント駆動、トレース駆動

- **解析的な方法** 待ち行列モデル、マルコフ連鎖モデル

- **M/M/1の待ち行列モデルの平均待ち時間** (利用率 / (1 - 利用率)) * 平均サービス時間

- **コンパイラ**
- 字句解析
- プログラムを表現する文字列を意味のある最小構成要素の単位（トークン）に分解する

- 構文解析
- 句解析で得られたトークンをもとに、言語の文法に基づいてプログラムを解析し、文法誤りがないか確認する

- 意味解析
- プログラムが意図的に正しいかを検証する。変数の宣言と使用を対応づけたり、演算におけるデータ型の整合性を確認する

- 最適化
- コードの実行効率を向上させるために、不要な演算の省略、ループの展開、関数のインライン展開、レジスタ割付などの変換を行う

- **オーバーレイ**
- プログラミングの命令が大きすぎて、主記憶に載らない時に少しずつ読み出す
- ストリームと似ていて、ストリームは命令ではなく、大量のデータを相手にしているのが違うが、メモリに少しずつ載せるという点では一緒

- クロスコンパイラ
- 異なるCPUアーキテクチャ向けに実行可能ファイルを生成するコンパイラ。

- ユニファイドメモリ方式
- CPUとGPUが同一の物理メモリを共有する方式。データ転送のオーバーヘッド削減に有効。

- ヘテロジニアスマルチコアプロセッサ
- 性能や省電力特性が異なるコア（例: 高性能コア + 省電力コア）を組み合わせたマルチコア。代表例はARM big.LITTLE。

- ホモジニアスマルチコアプロセッサ
- 全コアが同一仕様のマルチコア。処理分散や並列処理に有利。

- スーパースカラプロセッサ
- 1クロックで複数命令を並列実行可能なCPU設計。

- ソフトコアプロセッサ
- FPGA上に実装可能なプロセッサIP。用途に応じた柔軟な構成が可能。

### データベース

- **Trigger** ある行や列に対して特定の処理(INSERT, UPDATE, DELETE)が走ったことを契機として自動的に処理を行う仕組み、またはそのプログラムのこと。イメージとしては親テーブルにINSERTされたことをきっかけに子テーブルの情報も更新したりするってこと？Railsのフックで使ってそう

- **ロックの種類**
- **共有ロック** データの読み込みにつかうロックで他トランザクションによる更新処理ができなくなる

- **専有ロック** データを更新する時に使うロックで、他のトランザクションからの読み込み、更新ができなくなる

- 共有ロック中にしか共有ロック取れない、専有ロックはロックが何もない時にしか取れない

- **正規化**
- **第一正規化** 繰り返し項目や集合知を持つ属性を排除し、すべての属性が単一の値

- **第二正規化** 主キーの一部に関数従属する属性を別表に分離する

- **第三正規化** 非キー属性に関数従属する属性を別表に分離する

- **参照制約** 設定されているDBでは主キーを含む行が削除されたり、更新されたりすると、参照関係が損なわれるので、どのような形で参照整合性を回復するかを外部キー制約のON DELETEやON UPDATEに指定しておくことができる
- **CASCADE** 参照先のデータが更新、削除されると、対応する外部キーの値も自動的に更新、削除される

- **RESTRICT** 参照制約違反となるデータの更新、削除を拒否する

- **NO ACTION** 処理の終了時に参照整合性を確認し、違反していれば更新、削除を拒否する

- **SET NULL** 参照先のデータが更新、削除されると、対応する外部キーの値にNULLが設定される

- **SET DEFAULT** 参照先のデータが更新、削除されると、対応する外部キーの値にデフォルト値が設定される

- ロールフォワード
- チェックポイントからコミットを拾って被害前まで修復する

- コミットしてないとダメ、そらそう

- B+木インデックス
- 範囲検索に強いインデックス。RDBで多用される

- NOTでの検索は結局全件スキャンするので効率悪くなる

- 一致検索、範囲検索、値の重複が少ない（カーディナリティが高い）検索の場合に高速化が期待できる

- ハッシュインデックス
- 完全一致検索に強いインデックス。範囲検索には不向き

- WITH
- SQLの実行結果を一時的な表として名前付きで保存し、後続のSQLで使用する

- その際に自分自身を参照している場合はWITH RECURSIVEという再帰的なテーブルを作ることができる。以下のような構文

```sql
WITH RECURSIVE 仮想テーブル名(カラム1, カラム2, カラム3 ...) AS (
  <<非再帰的なSELECT>>
  UNION [ALL]
  <<再帰的なSELECT>>
)
```

- 文字列結合にはパイプを使用する
- LIKE ‘%:キーワード%’ → :キーワードが含まれるものを探してしまう

- LIKE ‘%’ `||` :キーワード `||` ‘%’ →こうすることで:キーワードの中身がbindされる、Excelみたいね

### 信頼性設計

| 用語 | 説明 | 具体例 |
| --- | --- | --- |
| フォールトアボイダンス (Fault Avoidance) | 設計や製造段階での品質向上により、そもそも故障や障害が発生しないようにする手法。 | 高品質部品の採用、厳格な設計審査。 |
| フォールトトレランス (Fault Tolerance) | 故障や障害が発生しても、システムが影響を最小限に抑えて動作を継続できる能力。 | 冗長化構成（RAID、クラスタリング）。 |
| フォールバック (Fallback) | 主系統が使えない時、代替手段や予備系統に切り替える仕組み。 | サーバ停止時にバックアップサーバへ自動切替。 |
| フェールセーフ (Fail-Safe) | 故障や異常が発生した際、危険を回避するため安全な状態に移行する設計。 | 電車のブレーキが故障時に自動で作動。 |
| フェールソフト (Fail-Soft) | 故障時に全停止せず、一部機能を制限して動作を継続する設計。 | サーバ障害時に性能を落としてもサービスを継続。 |
| フェールオペレーショナル (Fail-Operational) | 故障後もほぼ通常の機能を維持し、運転や作業を続ける設計。 | 航空機の多重操縦系統。 |
| フェイルファスト (Fail-Fast) | 異常が発生したらすぐに処理を停止し、問題を早期に顕在化させる設計。 | 入力データの不正を即時エラーで通知。 |
| フールプルーフ (Fool-Proof) | 人的ミスや誤操作をしても危険や損害を発生させない設計。 | USB端子を逆向きに挿せない構造。 |

### セキュリティ

- **認証技術**
- **チャレンジレスポンス方式** 通信系路上に固定パスワードを流さないようにすることで、盗聴によるパスワードの漏洩やリプレイアタックを防止する認証方式。サーバはクライアントから要求があるたびに異なる乱数値(チャレンジ)を生成して保持するとともに、クライアントへ送る。クライアントからは利用者が入力したパスワードとサーバから送られてきたチャレンジから所定の方法でレスポンスを計算する。クライアントは生成したレスポンスと利用者が入力した利用者IDをサーバに送る。サーバーはクライアントから受け取った利用者IDで利用者情報を検索して、取り出したパスワードと保持していたチャレンジを用いてクライアントと同じ手順でレスポンスを生成する。サーバは受け取ったレスポンスと生成したレスポンスが一致すれば認証成功

- **時刻同期式ワンタイムパスワード** トークンという装置が自動的に表示する、認証のたびに異なるデータをパスワードとして送信する

- **ナッシュ均衡** ゲームに参加する各プレイヤーが相手が選択すると予想される戦略に対して、自分の利得が最も大きくなる戦略（最適反応戦略）を選択している状態

- **マーケティング戦略**
- **4Pと4C**

| **4P (売り手視点)** | **4C (買い手視点)** |
| --- | --- |
| Product (製品) | Customer Value (顧客価値) |
| Price (価格) | Cost (コスト) |
| Place (流通) | Convenience (利便性) |
| Promotion (販売促進) | Communication (コミュニケーション) |

- **価格戦略**
- **プライスライニング戦略** 数段階の価格帯に分けて商品展開すること ハイロープライシング(2種類のみ)、松竹梅戦略(3種類のみ)などとも

- **キャプティブ戦略** 商品本体の価格を安く設定し、関連消耗品の販売で利益を得る戦略

- **ビジネスモデル**
- **フリーミアム** フリーとプレミアムを合わせた造語。基本は無料だけど、有料サブスクでは明確な差別化要素を提供（YouTubeのメンバーとかまさにそう）

- **指標**
- **コンバージョン率** 広告を経由してサイトに訪れたもののうち、どの程度が成約に至ったかを示す指標

- **ワントゥワンマーケティング** その名の通り一人一人に合わせたマーケティング活動でCRMシステムを活用する

### 企業と法務

- **会計・財務**
- **TCO（Total Cost of Ownership）** ある設備システムアンドを取得してから廃棄に至るまでに発生するコストの総額を表す、総所有コストとも。イニシャルコストとランニングコストからなる

- **損益分岐点売上高** = 固定費 / (1 - 変動比率) ※変動比率 = 変動費 / 売上高

- **国際会計基準**
- **IFRS（International Financial Reporting Standars）** 国際財務会計報告基準 世界共通で利用できる会計基準を目指して設定している基準

- **IASB（International Accounting Standards Board）** IFRSを設定している独立民間非営利の基準設定機関である国際会計基準審議会

- **SEC（Securities and Exchange Commission）** 米国証券取引委員会

- **GAAP（Generally Accepted Accounting Principles）** 企業会計原則

- **契約・法務**
- **レベニューシェア型契約** 収益を分け合うという名前の通り、アプリケーションの開発費を無償または安価にする代わり、そのアプリから得られる収益の何割かをSI事業者に支払う契約形態

- **パテントプール** 標準規格などに関して複数の特許権者が保有する特許を持ち寄り、一括で効率よくライセンスする仕組み

- **人材管理**
- **コンピテンシーモデル** 業務で高い成果を上げる人に凶つする行動特性や思考傾向をコンピテンシーとして抽出、体系化した人材評価モデル
