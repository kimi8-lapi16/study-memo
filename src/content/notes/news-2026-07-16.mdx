## 今日のTips: PostgreSQLの SELECT ... FOR UPDATE SKIP LOCKED でジョブキューを実装する

複数のワーカーが同じテーブルから未処理タスクを取り出す「ジョブキュー」的な処理では、素朴な<Mono>SELECT ... FOR UPDATE</Mono>だと他のワーカーがロック中の行に当たったときに待機が発生し、競合の原因になる。

PostgreSQL 9.5で追加された<Mono>SKIP LOCKED</Mono>オプションを使うと、即座にロックを取得できない行を結果からスキップできるため、複数ワーカーが同じキューテーブルに同時アクセスしてもロック待ちが発生しない。

```sql
SELECT * FROM job_queue
WHERE status = 'pending'
ORDER BY created_at
LIMIT 1
FOR UPDATE SKIP LOCKED;
```

<Tip label="▶ 使いどころに注意">
公式ドキュメントは「ロックされた行をスキップするため一貫性のないデータビューになり汎用の処理には向かないが、キューのようなテーブルへの複数コンシューマーアクセスにおけるロック競合の回避には使える」と明記している。集計処理など一貫性が必要な場面では使わないこと。
</Tip>

参考: [PostgreSQL: Documentation: SELECT](https://www.postgresql.org/docs/current/sql-select.html)
